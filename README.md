# Judgeman Bot (ジャッジマン・ボット)

「呪術廻戦」に登場する日車寛見の術式、領域展開**「誅伏賜死（ちゅうぶくしし）」**をコンセプトとした、法廷シミュレーションDiscord Botです。

> [!WARNING]
> **本ボットはジョークボットです。**
> サーバー内の小さなトラブルの解決や、身内での遊び、おふざけとして使用することを目的としています。深刻な対立や重要な意思決定には使用しないでください。

---

## Basic Commands
裁判を開始するには、以下のいずれかのコマンドを使用します。すべて同じ機能が起動します。
- `/judgement` (推奨)
- `/誅伏賜死`
- `/開廷`
- `/deadly-sentencing`

### 引数（パラメータ）
`/judgement` の後に続けて、以下の項目を指定します。
- **prosecutor (検事)**: 被告の罪を追及し、有罪を主張する当事者。
- **defendant (被告)**: 今回の裁判で罪を問われる当事者。
- **judge (裁判官)**: 当事者の主張を聞き、判決を下す権力者。
- **charge (罪状)**: 被告がどのような罪を犯したか（テキスト）。
- **defencer (弁護人)**: 被告の味方となり、罪を和らげる当事者。
  - ※空欄にした場合、被告自身が**被告兼弁護人**として扱われます。

## Workflow: Trial Progression

### 1. 召喚と審査
コマンドが実行されると、ジャッジマンが法廷（実行したチャンネル）に現れます。この際、以下の条件が自動でチェックされます。
- **オンライン確認**: 全ての当事者が「オンライン」であること（退席中・取り込み中は不可）。
- **活動確認**: 全ての当事者が過去1時間以内に、そのサーバーで発言していること。
- **重複確認**: 役割が重複していないこと（被告と弁護人の兼任は許可されます）。
- **クールダウン**: 実行者は前回使用から7日間経過していること（設定で無効化可能）。

### 2. 開廷の審査（妥当性投票）
ジャッジマンが提示した罪状に対し、1分間の投票が行われます。
- **被告人の重み**: 被告の「反対（妥当ではない）」は **3票分** としてカウントされます。
- **即時却下**: 被告以外の当事者（裁判官、検事、弁護人）が1人でも反対した場合、裁判はその場で直ちに却下されます。
- **可決条件**: 「賛成」票が「反対」票を上回った場合のみ、正式に開廷されます。

### 3. 公判（第1審）
裁判が開始されると、当事者に以下の変化が起こります。
- **専用ロールの付与**: **裁判官** (金), **検事** (赤), **弁護人** (青), **被告** (灰) のロールが付与されます。
- **ニックネームの変更**: 一時的に名前が `【役職】名前` の形式に変更されます。
  - ※元の名前は保存され、閉廷時に自動で復元されます。
- **発言制限**: 当事者は、裁判が行われているチャンネル以外での発言が禁止され、他チャンネルでの発言は自動削除されます。

裁判開始から5分後（テスト時は10秒）、裁判官に判決を下す権限が与えられます。

### 4. 判決と上告（再審）
- **有罪宣告 (`/gu`)**: 裁判官が有罪を宣告します。サーバー設定に応じた刑罰が適用されます。
- **無罪宣告 (`/in`)**: 裁判官が無罪を宣告します。刑罰なしで裁判が終了します。
- **再審 (`/re`)**: `/gu` による判決後30秒以内に、被告または弁護人が実行できます。最大で第3審まで行われ、再審のたびにペナルティ倍率が上昇します（2倍→4倍）。
- **陪審員投票**: 第2審以降は、裁判官ではなく「陪審員（当事者以外のユーザー）」の投票によって有罪・無罪が決定されます。

## Punishment Modes (Server Settings)
`/settingjudge` コマンドで、サーバーごとに判決時の刑罰モードを切り替えられます。この設定に応じて `/gu` コマンドの入力項目が自動的に変化します。

- **タイムアウトモード**: 指定した時間（1〜5分）のタイムアウトを物理的に付与します。
- **罰ゲームモード**: タイムアウトは行わず、裁判官が指定した「罰ゲーム（テキスト）」を課します。
- **なしモード**: 刑罰は与えず、有罪という名誉的（不名誉的）な宣告のみを行います。

※モードを切り替えると、ボットはそのサーバーのスラッシュコマンドを自動的に再登録し、最適な入力欄を表示するように調整します。

## Special Features
- **/dm [送信先] [内容]**: 当事者間でのみ利用できる秘匿通信です。
- **/action [内容]**: 法廷内での行動（「異議あり！」「資料を提出する」など）を宣言し、ログに記録します。
- **/info**: 現在進行中の裁判のステータスやPenalty倍率を表示します。
- **/forceclose**: 【開発者専用】トラブル時に裁判を強制終了し、全てのロール・名前を復元します。

## セットアップガイド

### 1. Discord Developer Portal での設定
1. アプリケーションを作成し、「Bot」セクションで **Privileged Gateway Intents** を全てオンにします。
   - `Presence Intent`
   - `Server Members Intent`
   - `Message Content Intent`
2. 以下の権限を付与してBotをサーバーに招待します。
   - `Manage Roles`, `Manage Nicknames`, `Moderate Members`, `Send Messages`, `View Channels`
3. **重要**: サーバー設定で **「ジャッジマン」のロールを、当事者となるユーザーのロールよりも上に配置** してください。そうしないと、名前の変更やタイムアウトが失敗します。

### 2. 環境変数 (.env)
プロジェクトのルートディレクトリに `.env` ファイルを作成し、以下の変数を設定してください。

```env
DISCORD_TOKEN=あなたのBotトークン
GUILD_ID=対象のサーバーID
CLIENT_ID=BotのクライアントID

# 裁判（領域展開）を許可するチャンネルID。カンマ区切りで複数指定可能。
# 空欄の場合はすべてのチャンネルで実行可能です。
ALLOWED_CHANNELS=123456789,987654321

# 裁判の開始（領域展開）の実行にのみ必要なロールID（任意）。
# 判決や再審などの裁判中の操作は、それぞれの役職に基づいて許可されます。
REQUIRED_ROLE_ID=

# 開発用：7日間のクールダウンを無効化する場合にtrueを指定。
DISABLE_COOLDOWN=true
```

### 3. 起動
```bash
npm install
node index.js
```
