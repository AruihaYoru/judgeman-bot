# judgeman-bot
「呪術廻戦」の日車寛見の術式、誅伏賜死を基としたDiscord Botです。

# 注意
**本ボットはジョークボットです。重要な問題などには使用せず、あくまで小さなトラブルが起こった際や面白半分、おふざけとして使用してください。**
**もちろんですが、その場にいないユーザーを役者にしないでください。**

## 基礎コマンド
もちろんですが、機能は一つのみです。
```text
/誅伏賜死
/開廷
/deadly-sentencing
/judgement
```
の4つがありますが、すべて同じ機能をスタートさせます。
本稿では、すべて`/judgement`で統一します。

## 引数
引数はすべて英語です。<br>  
形は`/judgement prosecutor:@username defendant:@username judge:@username charge:"text" defencer:@username `です。のように入力すると、領域展開「誅伏賜死」が開始されます！
<br>また、弁護人と被告人を同一人物にすることも可能です。その場合は、`defencer`の項に何も入力しないでください。
- **defencer（弁護人）：**ユーザー名を指定します。端的に説明するのならば、罪を軽くすることが役割です。
- **prosecutor（検事）：**ユーザー名を指定します。端的に説明するのならば、罪を重くすることが役割です。
- **defendant（被告）：**ユーザー名を指定します。罪を犯した人物を指します。
- **judge（裁判官）：**ユーザー名を指定します。弁護人と、検事のいうことを聞いて被告に罪を与えます。
- **charge（罪状）：**テキストを入力します。被告がどんな罪を犯したかを記述します。

## ワークフロー
`/judgement`を実行すると、ジャッジマンがその場に召喚されます。その際、以下の判定が行われます。
1. 召喚されたすべてのユーザーがオンラインであるか（ステータスが「オンライン」である必要があります）。また、1時間以内にそのサーバーで発言しているか。
2. defendant, prosecutor, judge, defencer で同じユーザーが重複して指定されていないか（ただし、defencerを省略した場合はdefendantが兼任します）。
3. 実行したユーザーは直近7日以内にこのコマンドを使用していないか（`.env`の`DISABLE_COOLDOWN`やテストモードで無効化可能）。
4. 詠唱者が`.env`に記載されたロールを所持しているか（設定されている場合）。

ジャッジマンが、その場に召喚された際、以下の投票を送信します。
```text
弁護人：@username
検事：@username
被告：@username
裁判官：@username

罪状：""
text
""

この裁判は妥当であるか？
```

### 開廷の承認ルール
1分間の投票が行われ、以下のルールで開廷かどうかが決定されます。
- **被告（Defendant）**の投票は、**反対なら3票分**、**賛成なら1票分**としてカウントされます。
- **被告以外の役者（裁判官、検事、弁護人）**が「反対」を選択した場合、その瞬間に裁判は即時却下されます。
- 最終的に**「賛成」の票数が「反対」の票数を上回った場合のみ**、開廷されます（同数や0票の場合は否決）。

```text
一定数以上の可決を得たため、これより裁判を執り行う。
...
```

### 裁判
裁判が開始されると、それぞれの役割に応じた専用ロールが付与されます。
- **裁判官** (金), **検事** (赤), **弁護人** (青), **被告** (灰) のように色分けされ、視認性が大幅に向上しました。

また、ニックネームが一時的に `【役職】名前` の形式に変更されます（例：`【裁判官】Username`）。
（※変更前の名前は自動で保存され、閉廷時に元の状態へ復元されます。）
裁判の開始から5分後（第1審の場合）、ジャッジマンが以下のメッセージを送信します：

```text
これより裁判官に、被告へ判決を下す事を許可する。
**/gu [timeout:分] [batsu:罰ゲーム内容]：** 有罪を宣告。タイムアウトと罰ゲームを指定可能。
**/in:** 無罪を宣告し、刑罰を課さない。

（※ /gu による判決が成された場合、弁護士・被告は一定時間内（30秒/テスト時15秒）に限り `/re` で再審可能だ）
```

### 判決と再審
- **有罪宣告（`/gu`）**：裁判官のみが実行可能。サーバー設定（`/settingjudge`）で許可されている範囲内（デフォルト1〜5分）でタイムアウトを付与できます。
- **再審（`/re`）**：判決後30秒以内に被告または弁護人が実行可能。最大で第3審（最終審）まで行われます。再審が行われるたびに、ペナルティ倍率（2倍→4倍）が上昇します。
- **陪審員投票**：第2審以降は、裁判官ではなく「陪審員（役職外のユーザー）」の投票によって有罪・無罪が決定されます。

### 設定と管理
- **/settingjudge**：サーバー内でのタイムアウト刑罰や罰ゲーム指定の有効/無効を切り替えます。
- **/resetsetting**：サーバー設定をデフォルトに戻します。
- **/forceclose**：【開発者専用】進行中の裁判を強制終了し、ロールやニックネームをリセットします。

### 裁判中の挙動・特別コマンド
- **/action [内容]**：法廷内での行動を宣言し、ログに記録します。
- **/dm [送信相手] [メッセージ内容]**：役者間での秘匿通信を行います。
- **/info**：現在進行中の裁判の詳細情報を表示します。

## その他
- すべての裁判ログは `log/` フォルダに保存されます。チャットログや投票記録、ユーザーIDの対応表が含まれます。
- 役者は、裁判が行われているチャンネル以外での発言が禁止（自動削除）されます。
- 裁判が終了すると、付与されたロールは自動的に削除され、ニックネームも復元されます。

## その他
また、すべての裁判ログは記録されます。`log/`フォルダに裁判ごとに保存されます。
その場合、
```text
裁判中のそのチャンネルでの発言録（傍聴人のチャットやコマンド実行アクション含む全て）
各役者
最終的な罰
詳細情報
```
が記録されます。<br>
`/info`を使用することで、実行中の裁判についての詳細をジャッジマンが返してくれます。これは、別のチャンネルでも使用することができます。

## 挙動
- 基本的に、JSONにリアルタイムで書き換えが入ります。
- 役者は**法廷（裁判が行われているチャンネル）以外での発言がシステム的に禁止（自動削除）**されます。

## セットアップ

自分のサーバーにこのDiscord Bot（ジャッジマン）を導入するための手順です。

### 1. Botの作成とトークンの取得
1. [Discord Developer Portal](https://discord.com/developers/applications) にアクセスし、新しいアプリケーションを作成します。
2. 左のメニューから「**Bot**」を開き、Botを追加します。
3. `Reset Token` ボタンを押して **Botのトークン** をコピーし、安全な場所に保存します。
4. 下にスクロールし、**Privileged Gateway Intents**（特権インテント）の以下をオンにして保存します。
   - **Server Members Intent** (ユーザー情報を取得およびニックネーム等の操作のため)
   - **Message Content Intent** (メッセージの内容を読み取り対応するため)
5. 左のメニューの「**OAuth2**」→「**URL Generator**」から、`bot` と `applications.commands` にチェックを入れます。
6. 追加で表示される「**Bot Permissions**」で、本Botの動作に必須となる以下のすべての権限にチェックを入れます。
   - **View Channels** (チャンネルを見る)
   - **Send Messages** (メッセージの送信)
   - **Read Message History** (メッセージ履歴を読む)
   - **Manage Roles** (ロールの管理：役職の付与/剥奪に必要)
   - **Manage Nicknames** (ニックネームの管理：裁判中の名前変更に必要)
   - **Moderate Members** (メンバーのタイムアウト等：刑罰の与奪に必要)
7. 生成されたURLをコピーしブラウザで開き、自身のサーバーにBotを招待します。

### 2. 環境変数の設定 (`.env`)
プロジェクトのルートディレクトリ（この `README.md` と同じ場所）に `.env` ファイルを作成し、以下の内容を記述してください。

```env
# Discord Developer Portalで取得したBotのトークン
DISCORD_TOKEN=ここにあなたのBotのトークンを入力してください

# Botを登録・使用する対象のサーバー（ギルド）のID (スラッシュコマンドの登録などに使用)
GUILD_ID=あなたのサーバーのIDを入力してください

# アプリケーションのクライアントID（Developer Portalの General Information から取得）
CLIENT_ID=あなたのクライアントIDを入力してください

# /judgement を実行可能な詠唱者（ユーザー）のロールID
# （@everyone など全ての人に許可したい場合、またはロールでの制限を設けない場合は空欄、もしくは対象のロールIDを指定）
REQUIRED_ROLE_ID=

# 開発者用設定：詠唱から7日間のクールダウンを無効化する場合にtrueを指定
DISABLE_COOLDOWN=true
```

### 3. インストールと起動
Node.js環境（v16.14.0以上推奨）がインストールされている状態で、ボットのディレクトリ内で以下のコマンドを実行します。

```bash
# 依存関係をインストール
npm install

# 自動起動設定（pm2など）
npm install pm2 -g
pm2 start index.js --name judgeman
```
※Botが意図せずクラッシュした際も、`pm2` 等を利用すれば自動でブートに入ります。

起動が成功すると、Discord上でジャッジマンがオンラインになり、領域が展開可能になります！# judgeman-bot
